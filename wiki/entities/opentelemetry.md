# OpenTelemetry

OpenTelemetry is an observability framework for collecting metrics, logs, and traces. It is used for distributed tracing in Project Chicken Soup.

## Key Features

- **Tracing** — Distributed tracing
- **Metrics** — Metrics collection
- **Logs** — Structured logging
- **Export** — Export to various backends
- **Multi-language** — Support for multiple languages

## Configuration

- **Collector** — OpenTelemetry collector
- **Exporter** — Export to Jaeger, Zipkin, etc.
- **Metrics** — Export to Prometheus
- **Logs** — Export to Loki

## Use Cases

- **Tracing** — Trace requests across services
- **Metrics** — Collect metrics
- **Logging** — Structured logging
- **Alerting** — Alert on anomalies

## Integration

- **Python** — Native Python API
- **FastAPI** — Integrate with FastAPI
- **Neo4j** — Trace Neo4j queries
- **LLM** — Trace LLM calls
- **Quantum** — Trace quantum circuits
