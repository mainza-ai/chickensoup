# Temporal Data Model

The temporal data model for Project Chicken Soup defines how time is represented in the data.

## Time Representation

- **Time as property** — Time is represented as a property of nodes
- **Time as dimension** — Time is a dimension in the graph
- **Time as node** — Time can be a node in the graph

## Branch Tracking

- **Many-Worlds** — When you travel back in time, you branch into a new timeline
- **Branch history** — Each branch has a history
- **Merge strategy** — Branches can be merged

## Causality

- **Causal relationships** — Causal relationships are preserved
- **Time travel** — Time travel preserves causality
- **Causal chains** — A causal chain is a sequence of events where each event causes the next
- **Temporal causality** — Causal relationships across time periods

## Information Processing

The temporal data model supports the flow of information through the system:

1. **Ingestion** — information is ingested from multiple sources
2. **Normalization** — information is normalized to a common format
3. **Fusion** — information is fused into a single coherent view
4. **Reasoning** — the system reasons about the fused information
5. **Output** — the system outputs timelines, destinations, and paths

## Integration

- **Neo4j** — Store temporal data in Neo4j
- **Qiskit** — Use Qiskit to simulate temporal evolution
- **PennyLane** — Use PennyLane for temporal quantum machine learning
- **Temporal Reasoning Engine** — Use the Temporal Reasoning Engine to reason about temporal data
