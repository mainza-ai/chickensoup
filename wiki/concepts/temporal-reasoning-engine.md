# Temporal Reasoning Engine

The Temporal Reasoning Engine is the core concept of Project Chicken Soup — it takes in information and produces timelines, destinations, and paths through spacetime.

## The Core Idea

The Temporal Reasoning Engine is a system that:

1. **Receives information** — witness accounts, sensor data, quantum measurements, documents
2. **Reasons about it** — determines temporal position, spatial position, causal relationships
3. **Produces outputs** — timelines, destinations, paths, evidence chains

## The Three Modes of Reasoning

### Temporal Reasoning
Reasoning about when things happened and how events relate across time. Includes:
- **Ordering events** — determining the sequence of events
- **Causal chain inference** — determining which events caused which
- **Evidence fusion across time periods** — combining evidence from different time periods
- **Temporal anomaly detection** — detecting unusual patterns in temporal data

### Spatial Reasoning
Reasoning about where things happened and how places relate across time. Includes:
- **Positioning** — determining the spatial coordinates of events
- **Field configuration** — determining the field configuration at each location
- **Path optimization** — finding the optimal path between locations

### Quantum Reasoning
Reasoning about the quantum state of spacetime. Includes:
- **Quantum state tomography** — reconstructing the quantum state of spacetime from measurements
- **Quantum state comparison** — comparing the quantum state of spacetime at different times
- **Quantum path optimization** — finding the optimal path through the quantum field

## How Information Flows

The system receives information in multiple formats:

1. **Witness accounts** — text with metadata (who, when, where, confidence)
2. **Sensor data** — numbers with coordinates (position, time, velocity, acceleration)
3. **Quantum measurements** — quantum states with metadata
4. **Documents** — papers, reports, articles with metadata
5. **Claims** — structured data with confidence scores

Each type of information is processed through a different pipeline:

- **Witness accounts** → Knowledge graph nodes → Temporal reasoning → Timeline
- **Sensor data** → Quantum state → Spatial reasoning → Destination
- **Quantum measurements** → Quantum state → Quantum reasoning → Path
- **Documents** → Knowledge graph → Causal reasoning → Evidence chain
- **Claims** → Knowledge graph → Bayesian reasoning → Confidence scores

## The Outputs

The Temporal Reasoning Engine produces four types of outputs:

1. **Timelines** — ordered sequences of events with temporal positions
2. **Destinations** — optimal locations in spacetime (x, y, z, t, field_configuration)
3. **Paths** — optimal routes through spacetime with energy, credibility, and time metrics
4. **Evidence chains** — causal chains of events with confidence scores

## The Connection to Quantum Algorithms

The Temporal Reasoning Engine uses several quantum algorithms:

- **Quantum Fourier Transform (QFT)** — transforms between position and momentum space
- **Quantum Phase Estimation (QPE)** — extracts eigenvalues of the spacetime Hamiltonian
- **QAOA** — finds the optimal time travel path through spacetime
- **VQE** — finds the ground state of the spacetime Hamiltonian
- **Quantum Annealing** — finds the global minimum of the spacetime energy landscape
- **Quantum Machine Learning** — trains the AI Navigator to recognize patterns

## The Connection to the Knowledge Graph

The Temporal Reasoning Engine uses the knowledge graph to:

- **Store information** — entities, relationships, properties
- **Query information** — find related entities, evidence, claims
- **Reason about information** — determine causal relationships, temporal positions, spatial positions

## The Connection to the Agent Architecture

The Temporal Reasoning Engine uses four agents:

- **Query Agent** — receives user queries, determines intent, routes to appropriate sub-agent
- **Research Agent** — explores the knowledge graph for evidence, credibility, and related claims
- **Navigation Agent** — runs the AI Navigator (PennyLane) to compute optimal paths through spacetime
- **Orchestrator Agent** — coordinates the flow: query → research → navigation → answer
